# 单人git 命令库——新手篇
事实上可以将被git在项目文件夹中管理的所有文件视为一个整体，把这个整体的状态称作一个“版本”
# 一、单分支工作流
## 1. git init
进入项目文件夹，然后执行：`git init`  
这个命令可以对该项目文件夹进行git仓库的初始化，标志该文件夹被加入git的管理  
在此过程中，创建git仓库的第一个分支，一般为**master分支**

## 2. git add .
创建文件之后，比如一个.py文件，不妨命名为first.py  
然后执行：`git add .`  
这个命令表示将此时文件夹中的所有文件放入**暂存区（staging area）**,为下面的提交做准备
(一般情况下，为了文件管理方便，最好不要单独add某一个文件，这是可以做到的，这设及git管理文件的方式，可以深入研究一下，在此不做赘述)

## 3. git commit -m "***" 
对于放在暂存区的文件，可以被提交到git仓库中  
只需要执行：`git commit -m "***"`  
其中`***`是你为这次提交自定义的名称    
这个命令可以将暂存区的内容正式的提交到本地的git仓库中
## 4. git status
如果你想看现在暂存区的内容，以及哪些被跟踪（tracked），哪些没有被跟踪（untracked），哪些有变化（modified），随时使用这个命令：
`git status`
## 5. git rm --cached
如果你想将文件从暂存区移除，进行更改（注意：文件一旦放到暂存区之后，再更改源文件，暂存区是不会同步变化的）之后再上传到暂存区的话需要用到这个命令：`git rm --cached`


# 二、多分支工作流
多分支工作的方式更为复杂，我们尝试用平行线理论来说明这个问题：  
创建新分支相当于创建一条新的平行线，每次commit都是在当前分支上向前前进一步，创建新分支意味着创建一条新的平行线，然后可以从当前文件的基础上垂直跨越到新平行线上继续向前走，又可以随时切换到任何别的分支上。  
这也表明，不同分支事实上是地位等价的，而最开始默认创建的名为master的分支只是主观意义上的主线
## 1. git branch `***`
当你想要在当前文件上创建新分支的时候（创建新的平行线）可以使用这个命令：`git branch ***`  
其中`***`同样是你自定义的分支名称
## 2. git switch `***`
创建分支之后还不行，你需要切换到你想去的分支上，此时可以执行这个命令:`git switch ***`,其中`***`就是你想转到的分支名

## 3. git branch
可以查看目前创建的所有分支,其中前面带*的分支为当前所在的分支

# 三、查找命令
## 1. git log --oneline --all
如果像查找某次提交的版本，可以使用：`git log --oneline --all`  
这个命令会罗列出历史提交，说明：
- 提交名的前面有7位数码为哈希值，相当于版本的身份证id,可通过哈希值追溯到对应的版本
- （）中的为分支名，提交是：按照提交时间向上生长，（）所在为分支同名指针所指版本，只要切换到对应分支，就会到达指针所指的版本，一般为最新版本
- HEAD指针所指的位置为当前所在分支
## 2. git checkout `***`
如果想回到特定版本，可以执行：`git checkout ***`  
其中`***`是对应版本的哈希值  
这个命令会分离HEAD指针，此时不在任何一个分支上：
- 第一种，看完就回到原处：直接switch到对应的分支即可
- 第二种，在此基础上创建新分支，就正常创建新分支，然后switch到新分支即可

# 四、和Github关联的个人开发命令
## 1. git remote add origin ***
其中
- ***是仓库的地址
- origin相当于你为这个仓库起的一个小名，可以自定义,一般习惯会叫origin
这就让本地和远程仓库建立了联系

## 2. git remote -v
这个命令可以查询到远程仓库的关联情况，因为一个项目可以关联到不止一个仓库（只要网址不同，小名不同就可以）
## 3. git push *** *** ：***
这就是把本地文件上传到GitHub仓库的命令
- 第一个***就是你为仓库起的小名
- 第二个***是你想推送的本地的it分支
- 第三个***是推送到GitHub仓库的那个分支，习惯上使用main分支
# 五、其他
## 1. git rebase -i --root
如果想要对某次提交改名，可以切换到对应分支,然后执行`git rebase -i --root`  
这个命令会从当前分支指针所在版本一路追溯到根版本（第一次提交），会罗列出过程中的所有版本  
这时有两个状态：  
- 一个是插入状态，按下i键就可以修改前面的状态，只需把pick改成reword即可
- 一个是命令行状态，这时打下：  
    - `:q!`会不保存更改，直接退出
    - `:wq!`会保存更改，然后退出，根据你的更改进入下一步操作

此时将pick改为reword之后就可以按ESC键退出插入状态进入命令行状态，输入`:wq!`就会进入改名的阶段，然后直接修改名称即可

## 2. git rebase -i HEAD~`*`
其中`*`可以填一个数字，表示从当前HEAD指针所在版本向前追溯`*`个版本，界面和git rebase -i --root一致，操作一致
## 3. git merge `***`
用于分支的融合，多用于并行开发不同模块时   
这里`***`是需要被合并的分支名，在融合之前需要先switch到“主线上”（你希望合并到的分支）  
对于分支被merge到master的过程会直接的改变master分支上的代码，此时在次commit一次就可以获得新版本的记录，上一次commit就是上一版
建议谨慎使用，防止原代码被破坏

## 4. .gitignore文件
这不是个具体命令，而是说在我们使用`git add .`命令的时候，有些内容并不需要使用git进行管理，可以在创建git仓库的相同文件夹内创建.gitignore文件
- 在git bash中可以使用`touch .gitignore`
- powershell中可以使用`New-Item .gitignore`

然后在该文件中列出不需要track的文件，一行一个即可,就是直接写文件名，有的时候可以使用通配符*，如果是文件夹则在末尾加上`/`即可
